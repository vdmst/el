# Панель управления учениками

Мини-приложение для менеджера онлайн-школы: список учеников, поиск, фильтры, карточка ученика, изменение статуса/комментария и создание нового ученика.

Репозиторий: https://github.com/vdmst/el
Деплой: https://vdmst.github.io/EL/#/

## Как запустить

1. Установить зависимости:

```bash
npm install
```

2. Запустить dev-сервер:

```bash
npm run dev
```

3. Открыть:

```text
http://localhost:3000
```

## Что реализовано

1. Страница списка учеников:

- таблица с полями `Id / Имя / Email / Курс / Статус / Дата регистрации`
- поиск по имени
- фильтр по статусу
- сортировка по дате регистрации
- переход в карточку ученика
- переход к форме создания

2. Карточка ученика:

- детальная информация (включая телефон, выполненные ДЗ, комментарий менеджера)
- изменение статуса
- изменение комментария
- сохранение изменений

3. Создание ученика:

- форма с полями `Имя / Email / Телефон / Курс / Статус`
- live-валидация полей при вводе
- проверка формата email
- проверка телефона (допустимые символы и длина 10–15 цифр)
- курс выбирается из списка предметов ОГЭ/ЕГЭ
- после успешного создания выполняется переход в карточку нового ученика

4. Дополнительно:

- фильтры списка синхронизированы с URL query params (`search`, `status`, `sort`)

## Технологии

- React 19
- TypeScript
- React Router
- Vite
- CSS
- Локальный mock API (fake async слой в памяти)

## Принятые решения

- Проект реализован в `src2` c тем же слоистым разбиением, что и в `src`:
    - `app` (entry, providers, router, store, layout, styles)
    - `pages` (страницы + lazy imports)
    - `widgets` (header, таблица учеников)
    - `features` (фильтры, создание, редактирование)
    - `entities` (модель ученика, API, UI-атом статуса)
    - `shared` (общие hooks/api/ui/config/lib/model)
- UI-элементы разнесены на отдельные модули (`Button`, `LinkButton`, `Input`, `Select`, `Textarea`) и универсальные form-field компоненты.
- Страницы сделаны максимально простыми: основная логика вынесена в `features/widgets`.
- Для API выбран локальный fake-слой с `Promise + setTimeout`, чтобы показать асинхронные запросы, loading/error/empty состояния без внешнего сервера.
- Состояние фильтров хранится в URL через `useSearchParams`, что поддерживает sharable-ссылки и корректную навигацию назад/вперед.

## Что можно улучшить

1. Подключить реальный backend (или `json-server`/MSW) вместо in-memory mock.
2. Добавить unit/integration тесты (валидация формы, фильтры, сценарии ошибок).
3. Добавить конфигурируемую схему полей формы (JSON/DSL), чтобы формы можно было собирать декларативно.
4. Добавить уведомления (toast) и более подробную обработку ошибок API.
